# コミュニティプラットフォーム 技術選定

## 技術スタック一覧

| レイヤー | 技術 | 役割 |
|---------|------|------|
| 言語 | **TypeScript** | 全レイヤー共通の開発言語 |
| Web フロントエンド | **Next.js（React）** | Webアプリの画面構築 |
| スマホアプリ | **React Native** | iOS/Android クロスプラットフォーム開発 |
| バックエンド | **NestJS**（Node.js上で動作） | API・ビジネスロジック・リアルタイム通信 |
| データベース | **PostgreSQL** | データの永久保存 |
| キャッシュ / リアルタイム | **Redis** | 高速な一時データ・セッション管理 |
| リアルタイム通信 | **WebSocket**（NestJS Gateway） | チャット・通知などの双方向通信 |
| ORM | **Prisma** | DB操作・マイグレーション管理 |
| ファイルストレージ | **AWS S3 / Cloudflare R2** | 動画・画像・ファイルのアップロード先 |
| クラウドインフラ | **AWS** | サーバー・DB・ストレージ等のホスティング |

---

## 各技術の選定理由

### TypeScript（言語）
- JavaScript に「型」を追加した言語
- フロント・バック・モバイル全てで共通利用でき、型定義やバリデーションロジックを共有できる
- コードを書いている段階でミスを検出できるため、品質が向上する

### Next.js / React（Webフロントエンド）
- **React**: 画面をコンポーネント（部品）単位で組み立てるUIライブラリ
- **Next.js**: Reactを拡張したフレームワーク。ルーティング、SSR（サーバーサイドレンダリング）、画像最適化、API Routes などを標準提供
- SSR対応によりSEOに強い

### React Native（スマホアプリ）
- iOS / Android を1つのコードベースで開発可能
- Web側と同じ TypeScript / React の知見を共有できるため、学習コスト・開発コストを削減

### NestJS（バックエンド）
- Node.js 上で動作するバックエンドフレームワーク
- モジュール構造が明確で、機能が多い本プロジェクト（12以上の機能モジュール）に適している
- 認証、バリデーション、WebSocket が標準で組み込まれている
- Express を選ばなかった理由：Expressは自由度が高い反面、大規模プロジェクトではコードが整理しづらい

### PostgreSQL（データベース）
- 無料かつ高機能なリレーショナルデータベース
- ユーザー × コミュニティ × ロール × ポイント × 注文など、複雑なリレーションの管理に最適
- JSON型カラムもサポートしており柔軟
- Stripe / Komoju 連携でのトランザクション処理に強い
- 全文検索にも対応可能（掲示板・メンバー検索）
- MySQL を選ばなかった理由：本プロジェクトのデータ構造が複雑であり、PostgreSQL の方が高度なクエリやデータ型に対応できる
- Oracle を選ばなかった理由：ライセンス費が高額（年間数百万〜数千万円）であり、本プロジェクトの規模では PostgreSQL で十分

### Redis（キャッシュ / リアルタイム）
- メモリ上にデータを保持する超高速データストア（読み書き 0.1ms以下）
- 用途：セッション管理、未読メッセージ数、WebSocket接続管理、イベント残り枠、APIキャッシュ

### WebSocket（リアルタイム通信）
- サーバーとブラウザが常時接続し、双方向でリアルタイム通信を行う仕組み
- NestJS の `@WebSocketGateway` で標準サポート
- 用途：チャット、未読バッジ通知、新着情報の即時表示、イベント参加枠のリアルタイム更新
- Firebase を選ばなかった理由：既にNestJS + PostgreSQL のバックエンドがあるため、Firebase を追加するとDB が二重管理になり複雑化する

### Prisma（ORM）
- TypeScript との相性が良く、型安全なDBクエリが書ける
- スキーマ定義からDB構造と TypeScript の型を自動生成
- マイグレーション（DBスキーマの変更管理）が容易

### AWS（クラウドインフラ）
- サーバー、DB、ストレージ、200以上のサービスを提供する総合クラウド
- PostgreSQL も MySQL も AWS 上で利用可能（RDS）
- EC・決済・動画・アナリティクスなど多機能な本プロジェクトに対応できる拡張性
- Firebase を選ばなかった理由：Firebase はアプリ開発特化で手軽だが、本プロジェクトのように画面55個・EC・決済・ポイントなど多機能なプラットフォームでは機能が不足する

## アーキテクチャ概要

```
┌─ クライアント ──────────────────────────────────────┐
│                                                    │
│  Next.js（Web）          React Native（iOS/Android）│
│                                                    │
└────────┬──────────────────────────┬─────────────────┘
         │ HTTP通信                  │ WebSocket
         │（通常のリクエスト）         │（リアルタイム）
         ▼                          ▼
┌─ バックエンド ──────────────────────────────────────┐
│                                                    │
│  NestJS（Node.js上で動作）                          │
│    ├── REST API（通常のデータ取得・更新）             │
│    └── WebSocket Gateway（チャット・通知）           │
│                                                    │
└────────┬──────────────────────────┬─────────────────┘
         │                          │
         ▼                          ▼
┌────────────────┐        ┌────────────────┐
│  PostgreSQL    │        │  Redis         │
│  （永久保存）    │        │  （高速一時データ）│
│                │        │                │
│  ・ユーザー情報  │        │  ・セッション    │
│  ・イベント     │        │  ・未読数       │
│  ・掲示板投稿   │        │  ・オンライン状態 │
│  ・注文・決済   │        │  ・キャッシュ    │
│  ・ポイント履歴  │        │                │
│  ・チャット履歴  │        │                │
└────────────────┘        └────────────────┘

         │
         ▼
┌────────────────┐
│  AWS S3 / R2   │
│  （ファイル保存） │
│                │
│  ・動画        │
│  ・画像        │
│  ・添付ファイル  │
└────────────────┘
```

## 補足：技術の関係性

```
TypeScript ─── 全体で使う「言語」
  │
  ├── ブラウザ側
  │     └── Next.js（フレームワーク）
  │           └── React（UIライブラリ）
  │
  ├── スマホ側
  │     └── React Native
  │
  └── サーバー側
        └── Node.js（実行環境）
              └── NestJS（フレームワーク）
                    ├── Prisma → PostgreSQL
                    ├── WebSocket Gateway → Redis
                    └── AWS SDK → S3
```
