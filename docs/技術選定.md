# コミュニティプラットフォーム 技術選定

## 技術スタック一覧

| レイヤー | 技術 | 役割 |
|---------|------|------|
| 言語 | **TypeScript** | 全レイヤー共通の開発言語 |
| Web フロントエンド | **Next.js（React）** | Webアプリの画面構築 |
| バックエンド | **NestJS**（Node.js上で動作） | API・ビジネスロジック・リアルタイム通信 |
| データベース | **Supabase**（PostgreSQL） | マネージドDB・pgroonga対応 |
| キャッシュ / リアルタイム | **Redis** | 高速な一時データ・セッション管理 |
| リアルタイム通信 | **WebSocket**（NestJS Gateway） | チャット・通知などの双方向通信 |
| ORM | **Prisma** | DB操作・マイグレーション管理 |
| ファイルストレージ | **Cloudflare R2** | 動画・画像・ファイルの保存・配信 |
| ジョブキュー | **BullMQ** | 非同期処理・バッチジョブ管理 |
| 画像処理 | **Sharp** | サムネイル生成・画像リサイズ |
| バリデーション | **Zod** | フロント・バック共通の入力検証 |
| UIライブラリ | **Tailwind CSS + shadcn/ui** | UIコンポーネント・スタイリング |
| データフェッチ | **TanStack Query** | API通信・キャッシュ管理 |
| 認証 | **bcrypt + @nestjs/jwt** | パスワードハッシュ・JWT認証（MVP: メール認証のみ） |
| メール配信 | **Resend** | トランザクションメール・一括配信 |
| エラー監視 | **Sentry** | エラートラッキング・パフォーマンス監視（Slack通知） |
| 日本語全文検索 | **pgroonga**（PostgreSQL拡張） | 掲示板・商品・メンバーの日本語検索 |
| LINE連携 | **LINE Messaging API** | プッシュ通知・メッセージ送信 |
| フロントエンドホスティング | **Vercel** | Next.js のデプロイ・CDN配信 |
| バックエンドホスティング | **Railway** | NestJS・Redis のデプロイ |

## 各技術の選定理由

### TypeScript（言語）
- JavaScript に「型」を追加した言語
- フロント・バック全てで共通利用でき、型定義やバリデーションロジックを共有できる
- コードを書いている段階でミスを検出できるため、品質が向上する

### Next.js / React（Webフロントエンド）
- **React**: 画面をコンポーネント（部品）単位で組み立てるUIライブラリ
- **Next.js**: Reactを拡張したフレームワーク。ルーティング、SSR（サーバーサイドレンダリング）、画像最適化、API Routes などを標準提供
- SSR対応によりSEOに強い

### NestJS（バックエンド）
- Node.js 上で動作するバックエンドフレームワーク
- モジュール構造が明確で、機能が多い本プロジェクト（12以上の機能モジュール）に適している
- 認証、バリデーション、WebSocket が標準で組み込まれている
- Express を選ばなかった理由：Expressは自由度が高い反面、大規模プロジェクトではコードが整理しづらい

### Supabase（データベース）
- マネージドPostgreSQLを提供するBaaS。本プロジェクトではデータベースのみ利用
- pgroonga 拡張をダッシュボードからワンクリックで有効化でき、日本語全文検索に対応
- Prisma からの接続がそのまま可能（接続URLを設定するだけ）
- 無料プラン：DB 500MB、無制限APIリクエスト
- PostgreSQL を選んだ理由：複雑なリレーション管理、JSON型、トランザクション処理、全文検索に対応
- MySQL を選ばなかった理由：本プロジェクトのデータ構造が複雑であり、PostgreSQL の方が高度なクエリやデータ型に対応できる

### Redis（キャッシュ / リアルタイム）
- メモリ上にデータを保持する超高速データストア（読み書き 0.1ms以下）
- 用途：セッション管理、未読メッセージ数、WebSocket接続管理、イベント残り枠、APIキャッシュ

### WebSocket（リアルタイム通信）
- サーバーとブラウザが常時接続し、双方向でリアルタイム通信を行う仕組み
- NestJS の `@WebSocketGateway` で標準サポート
- 用途：チャット、未読バッジ通知、新着情報の即時表示、イベント参加枠のリアルタイム更新
- Firebase を選ばなかった理由：既にNestJS + PostgreSQL のバックエンドがあるため、Firebase を追加するとDB が二重管理になり複雑化する

### Prisma（ORM）
- TypeScript との相性が良く、型安全なDBクエリが書ける
- スキーマ定義からDB構造と TypeScript の型を自動生成
- マイグレーション（DBスキーマの変更管理）が容易

### BullMQ（ジョブキュー）
- Redis 上で動作する非同期ジョブキューライブラリ
- NestJS 公式の `@nestjs/bullmq` で統合可能
- 用途：メール送信キュー、ポイント失効バッチ、HLS動画変換、カウンター集計
- 既に Redis を採用しているため追加インフラコストなし

### Sharp（画像処理）
- Node.js で最も高速な画像処理ライブラリ（libvips ベース）
- サムネイル生成、画像リサイズ、フォーマット変換（WebP等）に対応
- Next.js の画像最適化でも内部的に使用されており、エコシステムとの親和性が高い

### Zod（バリデーション）
- TypeScript ファーストのスキーマ定義・バリデーションライブラリ
- フロント（Next.js）とバック（NestJS）で同一のバリデーションルールを共有可能
- Prisma の型と組み合わせることで、DB → API → フロントまで型安全な開発ができる

### Tailwind CSS + shadcn/ui（UIライブラリ）
- **Tailwind CSS**: ユーティリティファーストのCSSフレームワーク。クラス名で直接スタイリングするため開発速度が速い
- **shadcn/ui**: 高品質なUIコンポーネント集。ソースコードが手元にコピーされるため、デザインのカスタマイズ性が高い
- Next.js の Server Components に対応
- MUI を選ばなかった理由：バンドルサイズが大きく、Material Design のスタイルに縛られやすい。独自デザインの自由度が低い

### TanStack Query（データフェッチ）
- サーバー状態管理の定番ライブラリ（旧 React Query）
- APIレスポンスのキャッシュ、自動再取得、楽観的更新、ローディング/エラー状態管理を標準装備
- SWR を選ばなかった理由：TanStack Query の方がミューテーション管理・キャッシュ無効化の機能が充実している

### bcrypt + @nestjs/jwt（認証）
- **bcrypt**: パスワードを不可逆なハッシュ値に変換して安全にDBに保存
- **@nestjs/jwt**: NestJS 公式の JWT モジュール。ログイン後のトークン発行・検証を担当
- MVP ではメール + パスワード認証のみ対応
- ソーシャルログイン（Google/Apple/LINE）は後から Passport.js を追加して対応可能
- NextAuth.js を選ばなかった理由：NestJS がバックエンドとして存在する構成では、認証を NestJS 側に統一した方がAPI保護がシンプルになる

### Resend（メール配信）
- モダンなメール配信API。開発体験が良く、TypeScript SDK が充実
- React Email と組み合わせることで、Reactコンポーネントとしてメールテンプレートを作成可能
- 月3,000通まで無料。小〜中規模のコミュニティに十分
- SendGrid を選ばなかった理由：無料枠が月100通と少なく、管理画面がやや複雑

### pgroonga（日本語全文検索）
- PostgreSQL の拡張機能。Groonga ベースの高速全文検索エンジン
- 日本語の形態素解析に対応しており、「東京タワー」で「東京」も「タワー」もヒットする
- Supabase のダッシュボードからワンクリックで有効化可能
- 追加コストなし。別途 Elasticsearch 等のサーバーを立てる必要がない
- 用途：掲示板投稿・商品・メンバー・イベントの検索

### LINE Messaging API（LINE連携）
- LINE公式のメッセージ送信API。ユーザーへのプッシュ通知・メッセージ送信に使用
- LINE Login は現時点では対象外（通知・メッセージ送信のみ）
- 無料メッセージ：月200通。有料プランで上限拡張可能
- Node.js 用の公式SDK（`@line/bot-sdk`）あり

### Sentry（エラー監視）
- エラートラッキング・パフォーマンス監視の定番サービス
- Next.js（Vercel）・NestJS（Railway）の両方に公式SDKがあり、フロント・バック両方のエラーを一元管理
- スタックトレース、エラー発生頻度、影響ユーザー数などを自動集計
- Slack 連携でエラー発生時にリアルタイム通知
- 月5,000イベントまで無料。小〜中規模のプロジェクトに十分

### Vercel（フロントエンドホスティング）
- Next.js の開発元が提供するホスティングサービスで、最も相性が良い
- Git push でデプロイ、プレビュー環境の自動生成、CDN配信が標準
- SSR / ISR / Edge Functions をそのまま活用可能

### Railway（バックエンドホスティング）
- NestJS・Redis をデプロイ・管理できるPaaS（PostgreSQL は Supabase に配置）
- Git push でデプロイ可能。環境変数管理・ログ閲覧がダッシュボードで完結
- WebSocket に対応しており、チャット・通知機能に支障がない
- Vercel + Railway は Next.js + NestJS 構成の定番の組み合わせ

## 補足：技術の関係性

```
TypeScript ─── 全体で使う「言語」
  │
  ├── ブラウザ側
  │     └── Next.js（フレームワーク）
  │           ├── React（UIライブラリ）
  │           ├── Tailwind CSS + shadcn/ui（スタイリング）
  │           ├── TanStack Query（データフェッチ）
  │           └── Zod（バリデーション）
  │
  └── サーバー側
        └── Node.js（実行環境）
              └── NestJS（フレームワーク）
                    ├── Prisma → Supabase (PostgreSQL)
                    ├── BullMQ → Redis（ジョブキュー）
                    ├── WebSocket Gateway → Redis
                    ├── bcrypt + JWT（認証）
                    ├── Sharp（画像処理）
                    ├── Zod（バリデーション）
                    ├── Resend（メール配信）
                    ├── LINE Messaging API（通知）
                    ├── Sentry（エラー監視 → Slack通知）
                    └── S3 API → Cloudflare R2
```
